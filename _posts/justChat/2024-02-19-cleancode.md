# <br>클린코드



## 01.나쁜 코드

> **깨진 유리창 법칙**
>
> 나쁜 코드는 깨진 유리창처럼 계속 나쁜 코드가 만들어 지도록 한다.

- 성능이 나쁜 코드
  - 불필요한 연산이 들어가서 개선의 여지가 있는 코드
- 의미가 모호한 코드
  - 이해하기 어려운 코드
  - 네이밍과 그 내용이 다른 코드
- 중복된 코드
  - 비슷한 내용인데 중복되는 코드들은 버그를 낳는다.
